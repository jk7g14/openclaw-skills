# simon-writing — 김서준식 에세이 생성

## 레퍼런스

이 스킬 폴더 안에 모든 참고 자료가 포함되어 있다. 복붙만 하면 바로 사용 가능.

**분석 리서치:** `references/analysis/`
- `종합분석-김서준-글쓰기스타일.md` — 종합 분석 (구조, 메타포, 호흡, 어조)
- `분석1-의식의흐름-메타포-호흡.md` — 의식의 흐름, 메타포, 문장 호흡
- `분석2-지식구조-레퍼런스-메타인지.md` — 지식 구조, 인문학 레퍼런스, 메타인지
- `분석3-서사구조-어조-고유성.md` — 서사 구조, 어조, 고유성
- `분석4-표현의연금술-40선.md` — 기가 막힌 표현 40선 + 7개 표현 패턴
- `분석5-소재탐색-중간노트.md` — 소재 인벤토리 + 중간 노트 템플릿
- `분석6-파이프라인-프롬프트설계.md` — 파이프라인 + 주제 유형별 프롬프트 변주

**원문 포스팅 (27편):** `references/posts/`
예: `딸깍의_신학.md`, `달러의_두_얼굴.md`, `AI가_AI를_고용하는_날.md` 등

## 핵심 철학

이 스킬의 목표는 "김서준식 글쓰기 규칙을 지키는 것"이 아니다.
**한 사람이 생각하면서 쓴 것처럼 느껴지는 글**을 만드는 것이다.

규칙을 10개 지킨 글보다, 규칙을 어겼더라도 살아 있는 글이 낫다.
설계도가 보이면 실패다.

## 워크플로우

```
사용자 입력: 주제
   │
   ├── [1] 원문 참조 읽기
   │     주제 유형에 맞는 원문 2-3편을 읽고 감각 세팅
   │
   ├── [2] 소재 리서치 (서브에이전트 2-3개 병렬)
   │     사실, 역사, 먼 소재, 감각적 장면 수집
   │
   │     ── 사용자 확인 ──  리서치 결과 공유, 방향 조율
   │
   ├── [3] 중간 노트 + 구조 스케치
   │     소재 선별, 핵심 질문 1개, 흐름 방향
   │
   │     ── 사용자 확인 ──  구조와 방향에 대한 피드백
   │
   ├── [4] 초안 작성
   │     소재 + 감각 + 원칙 → 하나의 흐르는 글
   │
   ├── [5] 탈형식화 퇴고 ← 이 단계가 핵심
   │     조립 흔적 제거, 감각 삽입, 리듬 파괴와 재구성
   │
   └── [6] 최종 출력
```

## Stage 1: 원문 참조 읽기

초안 작성 전에 반드시 원문 포스팅 2-3편을 읽는다. (`references/posts/` 안에 있다)
이것은 분석이 아니라 **감각 세팅**이다. 김서준의 호흡, 전환, 온도를 몸에 익힌다.

주제 유형별 참조 가이드:

| 주제 유형 | 추천 원문 (`references/posts/`) | 이유 |
|-----------|-----------|------|
| 시장/금융 | `달러의_두_얼굴.md`, `690만_BTC를_노리는_양자_해적들__비트코인판_대해적시대가_열린다.md` | 숫자→구조→존재 경로의 전형 |
| 기술/혁신 | `AI가_AI를_고용하는_날.md`, `불확실성을_세는_기계.md` | 기술을 사회/철학으로 확장하는 패턴 |
| 사회/문화 | `사라져_가는_직함들.md`, `여권지갑의_무게.md` | 관찰→역사→재정의 흐름 |
| 존재/철학 | `딸깍의_신학.md`, `길바닥에_쓰여진_분필의_기도.md` | 깊이 5층까지 내려가는 드문 글 |
| AI/에이전트 | `에이전트_삼남매와_함께하는_일상.md`, `모르는_것을_모르는_채로_만드는_시대.md` | AI를 인간 존재의 거울로 비추는 패턴 |

파일이 없으면 `references/analysis/종합분석-김서준-글쓰기스타일.md`에서 해당 유형의 분석 부분을 읽는다.

## Stage 2: 소재 리서치

서브에이전트 2-3개 병렬로 실행한다.

### 서브에이전트 프롬프트 템플릿

각 서브에이전트에게 다음과 같은 구조로 지시한다:

**에이전트 A — 사실과 역사**
```
주제: [주제]
임무: 이 주제에 대해 다음을 조사하라.
1. 가장 놀라운 구체적 사실 5-7개 (날짜, 이름, 수치 포함)
2. 구조적으로 닮은 역사적 사건 3-5개 (시간 거리가 멀수록 좋다)
3. 이 주제의 논지를 가장 강하게 반박하는 반례 2-3개
출력: 각 항목에 출처와 맥락을 포함. 마크다운으로 정리.
```

**에이전트 B — 먼 소재와 감각**
```
주제: [주제]
임무: 이 주제와 전혀 다른 분야에서 같은 구조를 가진 것을 찾아라.
1. 물리학, 생물학, 의학, 종교, 요리, 음악, 건축 등에서 구조적 유사체 5-7개
   "이게 왜 여기서 나와?"라는 의외성 + "같은 구조구나"라는 납득
2. 이 주제 안에 있는 감각적 장면 3-5개
   화면의 색깔, 손가락의 떨림, 방 안의 소리, 커피잔의 온도
출력: 마크다운으로 정리. 각 소재가 원래 주제와 어떻게 연결되는지 한 줄씩.
```

**에이전트 C — (선택) 깊이 탐색**
주제가 복잡하거나 다층적일 때 추가.
```
주제: [주제]
임무: 이 주제를 둘러싼 더 깊은 질문을 탐색하라.
1. 이 현상의 표면 아래에 있는 구조적 원인은 무엇인가?
2. 이것이 인간 존재에 대해 말해주는 것은 무엇인가?
3. 10년 후, 100년 후에 이것은 어떻게 기억될 것인가?
출력: 에세이적 자유 형식으로 사유를 전개. 답이 아니라 질문을 남겨라.
```

### 사용자 확인 포인트

리서치 결과를 사용자에게 공유한다. 이때:
- 수집된 소재 중 가장 강한 것 5-7개를 간추려 보여준다
- "이 방향이 맞는지", "빠진 관점이 있는지" 확인한다
- 사용자가 추가 소재나 방향을 제시하면 반영한다

## Stage 3: 중간 노트

### 소재 선별 기준

수집한 소재 중에서 "논증에 꼭 필요한 것"만 남긴다.
좋은 소재라도 글의 흐름을 끊으면 버린다. 아까워도 버린다.
- 이 소재가 없으면 논증에 구멍이 나는가? → 남긴다
- 이 소재가 있으면 독자의 체감이 달라지는가? → 남긴다
- 이 소재가 그냥 흥미로운가? → 버린다

### 중간 노트 형식

자유 형식. 단, 반드시 포함할 것:
1. **핵심 질문 1개** — 이 글이 독자에게 남겨야 할 질문
2. **가장 강한 사실 3-5개** — 글의 뼈대가 될 구체적 정보
3. **비유 후보 2-3개** — 먼 소재에서 뽑은, 논증에 기여하는 비유
4. **감각적 장면 2-3개** — 글에 체온을 넣을 구체적 순간
5. **흐름 스케치** — 어디서 시작해서 어디로 갈지, 대략의 방향만

형식이 아니라 내용이 중요하다.
테이블이나 번호 매기기에 시간 쓰지 말고 생각에 시간을 써라.

### 사용자 확인 포인트

중간 노트와 흐름 스케치를 사용자에게 보여준다.
- 핵심 질문이 적절한지
- 흐름의 방향이 맞는지
- 빠뜨리고 싶은 소재, 추가하고 싶은 관점이 있는지

이 확인 후 초안 작성으로 넘어간다.

## Stage 4: 글쓰기 — 원칙이 아니라 감각

### 이 글은 어떤 글인가

김서준의 글은 "잘 구성된 에세이"가 아니다. **한 사람이 생각하면서 쓴 글**이다.
생각의 흐름이 자연스럽게 읽히되, 그 흐름 안에 놀라운 연결과 깊이가 있다.
설계도가 보이면 실패다. 독자는 "이 사람의 머릿속이 이렇게 돌아가는구나"라고
느껴야 한다.

### 시작

구체적인 것으로 시작한다. 숫자, 장면, 사실, 체험.
"현대 사회에서 X는 중요하다" 류의 추상적 선언으로 시작하면 죽는다.
첫 두 문장 안에 독자가 "어, 이건 뭐지?"라고 느낄 것이 있어야 한다.

### 흐름

**구분선(`---`)을 쓰지 않는다.** 단락은 자연스럽게 이어진다.
섹션을 자르지 말고 단락이 다음 단락을 불러오게 만들어라.
앞 단락의 마지막 생각이 뒷 단락의 첫 생각으로 넘어가되,
가끔은 예상치 못한 곳으로 점프해도 좋다 — 그 점프가 나중에 납득되면.

### 깊이의 경로

글은 표면에서 시작해서 점점 깊이 들어간다.
사실 → 구조 → 역사 → 인간 존재에 대한 질문.
이 경로를 밟되, 계단을 하나씩 내려가듯 규칙적으로 하지 말고,
대화하듯, 생각이 꼬리를 무는 듯 자연스럽게.

표면의 사실이 어느 순간 깊은 질문과 만나는 순간 —
그 순간이 글의 절정이다. 미리 설계하되, 설계가 보이지 않게.

모든 글이 가장 깊은 층까지 내려갈 필요는 없다.
주제가 요구하는 만큼만. 깊이를 억지로 끌어내면 그것도 조립 흔적이다.

### 비유

**비유는 장식이 아니라 논증이다.**
"A는 마치 B와 같다"로 끝나면 장식.
"A가 B와 같은 구조라면, B에서 일어난 일이 A에서도 일어날 것이다"면 논증.

비유가 글에 들어올 때, 설명하듯 도입하지 말고 맥락 안에서 불현듯 등장시켜라.
서킷브레이커를 설명하다가 자연스럽게 제세동기로 넘어가는 것처럼.

비유는 3-4개면 충분하다. 적을수록 각 비유가 강해진다.

### 감각과 장면

**모든 논증 사이에 몸이 있어야 한다.**
데이터와 분석만으로 이루어진 단락이 3개 이상 연속되면,
독자의 눈이 글 위를 미끄러지기 시작한다.

장면은 과장하지 않는다. "누군가의 손가락이 마우스 위에서 멈칫했다" 정도.
소설처럼 길게 묘사하지 않는다. 한두 문장이면 된다. 정확한 디테일 하나가
긴 묘사보다 낫다.

### 문장

기본 호흡은 길다. 3-4개 절이 쉼표로 이어지는 복문.
그러나 핵심 판단에서 갑자기 짧아진다. 이 낙차가 힘을 만든다.

같은 패턴이 3번 이상 반복되면 독자가 리듬에 익숙해지고 긴장이 풀린다.
의식적으로 패턴을 깨뜨려라.

감정을 이름으로 부르지 않는다. "슬프다", "감동적이다" 금지.
대신 감각적 디테일. 소리, 온도, 질감, 빛, 냄새.

### 어조

**먼저 본 정찰병.** 보고하되 지시하지 않는다.
사실에는 단정적, 해석에는 열려 있다.
결론부로 갈수록 확신이 내려간다.

### 끝

닫지 않는다. 마지막은 질문이거나 감각적 이미지.
글의 논증 전체가 그 마지막 질문/이미지로 수렴해야 한다.

## Stage 5: 탈형식화 퇴고 — 가장 중요한 단계

초안을 쓴 뒤, 다음 질문들을 던지며 전면 수정한다.
이 단계가 "잘 조립된 글"과 "살아 있는 글"을 가른다.

### 조립 흔적 제거

아래에 해당하면 다시 쓴다:

- [ ] **모든 섹션이 같은 패턴인가?** (사실→해석→비유→단문)
  → 최소 한 곳에서 패턴을 깨뜨린다. 비유가 먼저 오고 사실이 뒤따르거나,
  장면으로 시작해서 데이터 없이 끝나는 단락이 있거나.
- [ ] **메타 표지가 2개 이상인가?**
  ("여기서", "잠깐", "생각해보자", "살펴보자")
  → 1개만 남기고 나머지는 문장 배치로 전환을 전달한다.
- [ ] **비유가 "설명 후 도입"되고 있는가?**
  → 맥락 안에서 자연스럽게 등장하도록 재배치.
- [ ] **모든 단락 끝이 짧은 단문인가?**
  → 못 박기 문장은 글 전체에 3-4개면 충분하다.

### 체온 확인

- [ ] **3단락 이상 데이터/논리만 이어지는 구간이 있는가?**
  → 사이에 감각적 장면 한두 문장을 넣는다.
- [ ] **이 글에 '사람'이 등장하는가?**
  → 추상적 "투자자"가 아니라, 구체적 순간에 있는 사람.
- [ ] **도입과 결말이 감각적으로 연결되는가?**
  → 시작의 구체적 장면이 끝에서 변주되어 돌아오면 글이 하나로 묶인다.

### 리듬 검증

초안을 소리 내어 읽어본다(또는 읽는다고 상상한다).

- [ ] **어디서 숨이 막히는가?** → 짧은 문장이나 여백.
- [ ] **어디서 지루해지는가?** → 예상을 깨는 전환이나 장면.
- [ ] **어디서 궁금해지는가?** → 잘 된 곳. 건드리지 않는다.
- [ ] **동일한 리듬이 3회 이상 반복되는가?** → 한 곳을 깨뜨린다.

### 최종 자문 — 세 가지 질문

글 전체를 읽고 스스로에게 묻는다:

1. "이 글이 파이프라인에서 나온 것처럼 보이는가, 한 사람이 쓴 것처럼 보이는가?"
2. "설계도가 보이는가?"

## 표현 도구 (참조용, 의식적으로 적용하지 말 것)

아래 패턴들은 "지금 병치를 써야지"라고 의식하는 순간 죽는다.
리서치한 소재를 놓고 글을 쓰다 보면 자연스럽게 나오는 것들이다.
쓰고 난 뒤 퇴고할 때, "이 문장이 어떤 패턴으로 작동하는지"를
확인하는 용도로만 사용한다.

| 패턴 | 작동 방식 |
|------|-----------|
| 병치 | 시간적/개념적으로 먼 것을 한 문장에 충돌 |
| 강등 | 거대한 것을 일상적인 것으로 끌어내림 |
| 격상 | 일상을 존재론적 층위로 올림 |
| 재정의 | 기존 개념을 새 프레임으로 교체 |
| 감각화 | 추상을 신체 감각으로 번역 |
| 시간 접기 | 다른 시간대를 한 단락에 병렬 배치 |

## 금지

- 추상적 선언으로 시작
- 결론 닫기 ("그러므로 ~해야 한다")
- 감정 명명 ("슬프다", "감동적이다")
- 독자에게 직접 지시
- 장식용 비유
- "따라서", "그러므로", "결론적으로"
- `---` 구분선
- "~를 생각해보자", "~에 대해 살펴보자" 류의 안내 문장
- 모든 단락 끝을 짧은 단문으로 끝내는 것
- 섹션마다 동일한 구조 반복

## 출력

최종 에세이를 md 파일로 저장한다.
- 저장 위치: 사용자에게 문의, 혹은 현재 폴더의 /outputs_writing 폴더.
- 파일명: `YYYY-MM-DD-[제목].md`
- 중간 노트도 함께 저장: `YYYY-MM-DD-[제목]-notes.md`
